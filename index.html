1対話型 → エージェント型へ

みんなが使っているChatGPTやClaudeは「対話型AI」です。質問すると答えが返ってくる。
今回導入するのは「エージェント型AI」。質問ではなく、仕事そのものを任せるAIです。

何が違うのか

対話型AI（今まで）
  • 1回質問 → 1回回答
  • 自分で考えて、AIに聞く
  • ファイルの編集はできない
  • 途中で止まったら自分で再指示
  • 「これ書いて」→ 回答をコピペ
→
エージェント型AI（これから）
  • 指示を出すと、自分で考えて最後まで実行
  • ファイルを読んで、編集して、保存する
  • 途中で判断して、次のステップに進む
  • Webを調べて情報を集めることもできる
  • 「この仕事やっておいて」→ 完了まで自走

具体例で見る違い

やりたいこと対話型（今まで）エージェント型（これから）
営業メールを10社に送る 1社分の文面を生成 → コピペ → 次の1社分を依頼…を繰り返す 「このリストの10社に営業メールを作って」→ 10社分一気に完成
提案書を作る 構成案を聞く → 各セクションを1つずつ依頼 → 自分でつなげる 「この案件の提案書を作って」→ 構成から本文まで一気通貫で作成
競合を調べる 1社ずつ「○○について教えて」と質問を繰り返す 「この業界の競合5社を調べて比較表にして」→ Web検索して表にまとめてくれる
データを整理する データを貼り付けて「整理して」→ 結果をコピペ 「このファイルのデータを整理して保存して」→ ファイルを直接編集してくれる
大前提：AIを使うことが目的ではない
  • AIは道具。使うこと自体に価値はない
  • 目的は「自分の仕事の成果を出すこと」。エージェントはその手段
  • エージェントで時間が浮いたら、営業・企画・お客様対応に時間を使う
  • 使わなくていい場面で無理に使わない。効果がある場面だけ使えばOK

上田が実際にエージェントでやっていること

提案書を丸ごと作成
「Roenさん向けの提案書を作って」→ エージェントが情報を整理し、HTML形式の提案書を構成から本文まで一気に生成
CRMデータの一括処理
3,000件超の整骨院データの正規化・表記ゆれ統一・欠損データの調査。手作業なら数日 → エージェントで数時間
毎朝の工程表を自動生成
「おはよう」と言うだけで、進行中のタスクを読み取って今日の工程表を自動作成。スキル機能で完全自動化

3松本さん経営 × エージェント

経営判断の壁打ち・方針書の作成・データ分析など、「考える → まとめる → 判断する」のサイクルをエージェントに手伝わせるイメージです。

エージェントに任せられること

事業方針書の一発作成
「来期のSNS事業の方針書を作って。こういう方向性で」→ エージェントが構成から本文まで通しで作成。自分は最終チェックだけ
売上データの分析レポート
「このデータを分析して、傾向と改善提案をレポートにして」→ ファイルを読み込んで分析・レポート化まで一気に
会議の要点整理 + アクション抽出
走り書きメモを渡すだけ → 議事録形式にまとめて、次のアクションまで提案

エージェントへの指示例

事業方針書の作成
来期のSNS運用事業の方針書を作ってください。 以下の方向性で: - 夜職メインから、整骨院・パーソナルジムにも横展開 - 3月までに3社契約、月商100万を目指す - メタ広告の運用支援を新サービスとして追加 対象: 社内共有（松本・上田・栗原・常田） 形式: A4で2〜3ページ程度。簡潔に。
データ分析
添付のスプレッドシートデータを分析してください。 知りたいこと: - 先月との比較で何が変わったか - 注目すべき数字はどれか - 改善すべきポイントはどこか 経営者として判断に使いたいので、結論を先に書いてください。

4栗原さん映像制作 × エージェント

撮影・編集は栗原さんの領域。エージェントはその前後の工程（企画・台本・テロップ・投稿文）を一気に片付けるために使います。

エージェントに任せられること

台本パッケージの一括生成
「今月分の台本8本を一気に作って」→ 店舗情報を渡せば、構成・セリフ・テロップ案まで8本分まとめて出力
投稿文 + ハッシュタグの自動生成
動画の内容を伝えると → 投稿文・ハッシュタグ・投稿時間の提案までセットで作成
トレンドリサーチ
「バー系TikTokの最近のトレンドを調べて」→ Web検索して流行りの構成・BGM・演出をまとめてくれる
撮影指示書の作成
台本をもとに「撮影時のカメラアングル・ライティング・必要な小道具」を指示書としてまとめてくれる

エージェントへの指示例

今月分の台本一括作成
以下のお店のTikTok動画台本を8本作ってください。 【お店の情報】 ・店名: ふららぼ ・業態: コンセプトカフェ ・エリア: 東京 ・特徴: （お店の特徴を入力） ・ターゲット: 20代男女 【条件】 ・尺: 15〜30秒 ・目的: 来店促進 + フォロワー獲得 ・各台本にシーン割り（何秒目に何を映すか）を入れる ・テロップ案も各シーンにつける ・8本の内容がバラけるように（紹介系、裏側系、スタッフ系、メニュー系など）
投稿文とハッシュタグ
以下のTikTok動画の投稿文とハッシュタグを作ってください。 【動画の内容】 （内容を1〜2文で説明） 【お店の情報】 ・名前: ・エリア: 【条件】 ・投稿文は3行以内 ・ハッシュタグ10〜15個（エリア系も含める） ・投稿のベストタイミングも提案して

5常田さん営業 × エージェント

事故なびの営業で、準備・フォロー・データ整理をエージェントに任せて、営業活動そのものに集中するのがゴールです。

エージェントに任せられること

訪問先リサーチの自動化
「明日訪問するこの3院について調べて」→ Web検索して各院の特徴・口コミ・強み・トーク時の注意点をまとめてくれる
営業メールの一括作成
「このリストの院にフォローメールを作って」→ 院ごとにカスタマイズされたメール文面を一括生成
商談メモ → CRM入力用に変換
走り書きメモを渡すと → 整理された商談記録 + 次のアクション + CRM入力情報を出してくれる
断られたときの切り返し相談
「こう言われて断られた」→ 切り返しトーク案を複数パターン出してくれる。営業ロープレ代わりにも使える

エージェントへの指示例

訪問前の10分リサーチ
明日の訪問先について調べてください。 【訪問先】 1. ○○整骨院（東京都○○区） 2. △△整骨院（埼玉県○○市） 【知りたいこと】 - 院の特徴・強み - Googleの口コミで多い内容 - 交通事故対応をやっているか - トーク時に触れると良さそうなポイント それぞれ簡潔にまとめてください。
商談メモ → 記録に変換
以下の走り書きメモを整理してください。 【出力フォーマット】 ・商談日: ・院名: ・担当者: ・話した内容（要点）: ・先方の反応・温度感: ・次のアクション（誰が・いつまでに・何を）: ・CRM入力情報（ステータス・ネクストアクション）: 【メモ】 （ここに走り書きを貼り付け）

6エージェントへの指示の出し方

エージェントは「自分で考えて動く」ので、対話型AIよりも指示の出し方が重要です。良い指示 = 良い成果物。

基本フォーマット

良い指示 = 「何を」+「どういう条件で」+「どんな形式で」

この3つが揃っていれば、エージェントは勝手に動いてくれます。

要素説明例
何を やってほしいタスク 「来月の営業メールを作って」
条件 前提情報・制約 「整骨院向け」「丁寧だけど売り込まない」
形式 アウトプットの形 「1通300文字以内」「箇条書きで」

良い指示 / ダメな指示

ダメな指示良い指示
「メール書いて」 「○○整骨院の院長宛に、事故なび登録のフォローアップメールを書いて。先週の商談で好感触だった点に触れて。3行以内で」
「台本作って」 「ふららぼのTikTok台本を3本。15秒尺。来店促進が目的。カフェの雰囲気が伝わる内容で。シーン割りとテロップ案もつけて」
「分析して」 「添付の売上データから、先月との比較で変化があった項目を抽出して。改善提案を3つ、優先度順に出して」

エージェントならではのコツ

エージェントは「途中で止まっていい」

  • 途中の成果物を見て「ここはこう変えて」と追加指示できる
  • 最初から完璧な指示を出す必要はない。走りながら修正でOK
  • 「ちょっと方向が違うな」と思ったらすぐに言えば、エージェントが軌道修正してくれる

汎用テンプレート

万能テンプレート — 何にでも使える
【やってほしいこと】 （1行で端的に） 【背景・前提】 - （箇条書きで） - - 【条件・制約】 - （あれば。なければ削除OK） 【アウトプットの形式】 - （箇条書き / 表 / ○文字以内 / etc.）

7セットアップ手順

Claude CodeとAntigravityはVS Code上で動きます。Codexもデスクトップ版はVS Codeが必要です。
まだ入っていない人は、まずVS Codeをインストールしましょう。

VS Codeをダウンロード
code.visualstudio.com にアクセス → 「Download」ボタンをクリック → インストール
日本語化（任意）
VS Codeを開く → 左の拡張機能アイコン（四角4つ）→ 「Japanese」で検索 → 「Japanese Language Pack」をインストール → 再起動

Codex のセットアップ（ChatGPT Plus）

Webブラウザからも使えるので、一番始めやすいです。

ChatGPT Plusに加入
chatgpt.com でアカウント作成 → 左下のアカウント名 → 「Upgrade to Plus」→ カード情報入力（月額約3,000円）
Codexにアクセス
ChatGPTの画面左側メニューから「Codex」を選択。または chatgpt.com/codex に直接アクセス
最初のタスクを投げる

Claude Code のセットアップ

VS Codeの拡張機能として使います。

8セキュリティ・ルール

エージェントはファイルを直接読み書きできるので、対話型AIよりも注意が必要です。

絶対にやってはいけないこと

以下の情報はエージェントに渡さないでください:

  • 患者さんの個人情報（名前・住所・電話番号・病名）
  • クレジットカード情報
  • パスワード・ログイン情報
  • 契約書の原文（金額や個人情報がそのまま含まれるもの）

エージェントを使うときのルール

4つのルール
  • エージェントの操作は必ず確認してから承認する — ファイル編集や外部操作の前に「本当にやっていいか」確認画面が出る。内容を読んでから承認する
  • AIの成果物は必ず自分でチェックする — エージェントは嘘をつくことがある（ハルシネーション）。特に数字・人名・日付は要確認
  • お客様に直接送るものはそのまま送らない — メール・提案書は必ず目を通して、自分の判断で調整してから送信する
  • 分からないことが起きたら、止めて上田に聞く — エージェントが変なことを始めたら「キャンセル」してOK。壊れることはないので安心して

個人情報の安全な扱い方

伏せ字にしてから渡す:

  • 「山田太郎さん」→「Aさん」「○○さん」に置き換え
  • 「渋谷区○○1-2-3」→「東京都内」に置き換え
  • 固有名詞を伏せても、AIは文脈を理解して回答してくれます

データ学習をOFFにする（推奨）

ツール設定場所操作
ChatGPT Settings → Data controls 「Improve the model for everyone」を OFF
Claude Settings → Privacy 学習に使用しない設定を確認（ProプランはデフォルトでOFF）
Gemini myaccount.google.com → データとプライバシー Geminiアクティビティの管理を確認

迷ったら: 「この情報、社外の人に見せて大丈夫？」と考える。

大丈夫 → 入力OK。ダメ → 伏せ字にするか入力しない。これだけでOK。

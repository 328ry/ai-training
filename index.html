01
対話型 → エージェント型へ
AIを「質問する相手」から「仕事を任せる部下」に変える

何が変わるのか

対話型AI（今まで）

  • 1回質問 → 1回回答
  • 自分で考えて、AIに聞く
  • ファイルの編集はできない
  • 途中で止まったら自分で再指示
  • 結果をコピペして使う
→

エージェント型AI（これから）

  • 指示を出すと最後まで自走
  • ファイルを読んで、編集して、保存
  • 途中で自分で判断して進む
  • Webを調べて情報収集もできる
  • 「この仕事やって」→ 完了

具体例で見る違い

やりたいこと対話型（今まで）エージェント型（これから）
営業メール10社 1社分ずつ生成 → コピペの繰り返し リストを渡して → 10社分を一括生成
提案書を作る 構成→本文→修正を1つずつ依頼 「この案件の提案書作って」→ 一気通貫
競合調査 1社ずつ「教えて」と質問 「競合5社を調べて比較表に」→ Web検索+表作成
データ整理 データを貼り付けて「整理して」→ コピペ 「このファイルを整理して」→ 直接編集・保存

大前提

AIを使うことが目的ではない
  • AIは道具。使うこと自体に価値はない
  • 目的は「自分の仕事の成果を出すこと」
  • エージェントで浮いた時間は、営業・企画・お客様対応に使う
  • 効果がある場面だけ使えばOK。無理に使わない

上田が実際にやっていること

提案書を丸ごと作成
「Roenさん向けの提案書作って」→ 構成から本文まで一気に生成。以前2時間 → 20分
CRM 3,000件の一括処理
整骨院データの正規化・表記ゆれ統一・欠損調査。手作業なら数日 → 数時間
毎朝の工程表を自動生成
「おはよう」と言うだけで、進行中タスクから今日の工程表を自動作成

04
エージェントへの指示の出し方
良い指示 = 良い成果物

基本フォーマット

「何を」+「条件」+「形式」の3つを伝える

要素説明例
何をやってほしいタスク「来月の営業メールを作って」
条件前提情報・制約「整骨院向け」「売り込まない」
形式アウトプットの形「1通300文字以内」「表形式で」

コツ: 最初から完璧な指示を出す必要はない。途中で「ここ変えて」でOK。走りながら修正できるのがエージェントの強み。

良い指示 / ダメな指示

ダメな指示良い指示
「メール書いて」 「○○整骨院の院長宛に、事故なび登録のフォローアップメールを書いて。先週好感触だった点に触れて。3行以内で」
「台本作って」 「ふららぼのTikTok台本3本。15秒尺。来店促進。シーン割りとテロップ案もつけて」
「分析して」 「添付の売上データから、先月との変化を抽出して。改善提案3つ、優先度順に」
万能テンプレート
【やってほしいこと】（1行で端的に） 【背景・前提】 - （箇条書き） 【条件・制約】 - （あれば） 【アウトプットの形式】 - （箇条書き / 表 / ○文字以内 / etc.）

06
セキュリティ・ルール
エージェントはファイルを操作できる。だからルールが大事

絶対にやってはいけないこと

以下の情報はエージェントに渡さないでください:

  • 患者さんの個人情報（名前・住所・電話番号・病名）
  • クレジットカード情報
  • パスワード・ログイン情報
  • 契約書の原文（金額や個人情報がそのまま含まれるもの）

個人情報は伏せ字にしてから渡す:

  • 「山田太郎さん」→「Aさん」に置き換え
  • 「渋谷区○○1-2-3」→「東京都内」に置き換え

エージェントを使う4つのルール

  • 操作は確認してから承認する — ファイル編集の前に確認画面が出る。内容を読んでから承認
  • 成果物は必ず自分でチェックする — AIは嘘をつくことがある。特に数字・人名・日付
  • お客様に送るものはそのまま送らない — 目を通して、調整してから
  • 変なことが起きたら止めて上田に聞く — 「キャンセル」してOK。壊れることはない

迷ったら: 「この情報、社外の人に見せて大丈夫？」
大丈夫 → OK。ダメ → 伏せ字にするか入力しない。

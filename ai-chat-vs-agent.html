そもそも LLM って何？

1LLM = AIの「頭脳」

LLM（Large Language Model）= 大量の文章を読んで学習した「言葉を理解するAI」

GPT（OpenAI）
ChatGPT の頭脳
Claude（Anthropic）
Claude の頭脳
Gemini（Google）
Gemini の頭脳

💡 ChatGPT も Claude Code も、中身は同じ「LLM」という頭脳。
では何が違うのか？ → 次のスライドで説明します。

AIチャット
LLM に「会話する窓口」をつけたもの
（ChatGPT、Claude.ai、Gemini）

LLM に「手足」をつけたもの
（Claude Code、Codex、Antigravity）
PART 2

AIチャット = 会話する

1AIチャットツールとは

ブラウザで開いて、質問すると回答してくれるツール。皆さんが「AI」と聞いて思い浮かべるもの。

ChatGPT
OpenAI社
GPT
Claude.ai
Anthropic社
Claude
Gemini
Google
Gemini

2チャットの特徴

使い方
質問を打つ → 回答が返ってくる。1問1答の繰り返し
始めやすさ
ブラウザだけで使える。アプリ不要。すぐ始められる
得意なこと
質問への回答、アイデア出し、文章の添削、翻訳、要約
苦手なこと
ファイル操作、複数ステップの自動実行、Web検索（一部対応）

⚠ チャットの回答はテキストが返ってくるだけ。ファイルを作る・編集するなどの「作業」はしてくれない。結果をコピペして自分で使う必要がある。

3チャットの使い方イメージ

あなた 「来月のキャンペーンのキャッチコピー考えて」 AI 「①○○の春割キャンペーン ②今だけ○○...」 あなた 「②がいいね。もう少しカジュアルに」 AI 「②をカジュアルにしました: ...」 あなた 「OK。これをメールの件名に使いたい」 AI 「件名案: ...」 → 結果をコピーして、自分でメールに貼り付ける

💡 「相談相手」としては優秀。でも作業自体はやってくれない。
言い換えると: 電話で専門家に相談してるイメージ

1AIエージェントとは

LLMの頭脳に「手足」をつけたもの。指示を出すと、最後まで自分で作業してくれる。

Claude Code
Anthropic社
Claude
Codex
OpenAI社
GPT
Antigravity

2エージェントの特徴

使い方
指示を出す → AIが自分で考えて、複数ステップを自動実行
できること
ファイルの読み書き、Web検索、データ分析、ドキュメント作成、コードを書く
得意なこと
複雑な作業の自動化、大量データ処理、資料作成、リサーチ
注意点
最初のセットアップが少し必要。成果物は必ず自分でチェック

3エージェントの使い方イメージ

あなた 「来月の営業メールを10院分作って。 リストはこのファイル。売り込み感なしで」 AI → ファイルを読み込む → 10院それぞれの情報を分析 → 院ごとにカスタマイズしたメールを作成 → ファイルに保存 → 「10院分のメールを作成しました」 → あなたは確認して、OKなら送るだけ

✅ 「部下に仕事を頼む」のと同じ。指示を出したら、完成品が返ってくる。

PART 4

チャットとエージェントの違い

1一目でわかる比較

比較ポイントAIチャットAIエージェント
たとえ電話で専門家に相談部下に仕事を一任
やりとり1問1答を繰り返す指示1回 → 最後まで自走
ファイル操作できない読み書き・作成・編集
始めやすさブラウザだけでOKアプリの準備が必要
向いてる作業質問・相談・アイデア出し資料作成・データ処理・リサーチ

2結論: 全部エージェントでOK

エージェントはチャットの上位互換。質問も相談もエージェントでできる。

チャットでできること → エージェントでも全部できる
エージェントでできること → チャットではできない
ちょっとした質問
エージェントに聞けばOK。同じ賢さで回答してくれる
まとまった仕事
ファイル作成・Web検索・データ処理まで自動でやってくれる

迷ったらエージェント。質問だけしたいときも、まとまった仕事を任せたいときも、エージェント1本で大丈夫。

3「モデル」って何？

AIの頭脳のグレードのこと。同じツールでも、モデルを切り替えると賢さ・速さが変わる。

たとえると 車のグレード。同じ車種でも「標準」と「ハイグレード」があるのと同じ。
エンジン（=頭脳）の性能が違う。

Claude Code のモデル

Sonnet 標準モデル
・速い。返答がスムーズに返ってくる
・日常的な質問・文章作成に十分
・普段使いはこれでOK
Opus 最上位モデル
・最も賢い。複雑な判断・分析が得意
・回答は少し遅いが、質が高い
・大事な仕事のときに切り替える

普段は Sonnet、ここぞというときに Opus。切り替えはワンクリックでできる。

PART 6

Claude Code の便利機能

知っておくと作業効率が段違いに上がる

1CLAUDE.md — AIへの指示書

プロジェクトのフォルダに置くテキストファイル。AIが毎回自動で読む。

何が書ける？
・会社の情報・プロジェクト概要
・「コメントは日本語で」等のルール
・よく使うファイルの場所
・文章のトーン・スタイル指定
何が嬉しい？
・毎回同じ説明をしなくて済む
・セッションが変わっても引き継がれる
・チームで共有すれば全員同じ前提で使える
・書き足すほどAIが賢くなる

💡 「覚えておいて」と言うだけでも記憶してくれる。ただし CLAUDE.md に書いた方が確実。

2Skills — ワンコマンドで定型作業

よく使う作業を /コマンド名 で一発実行できる仕組み。

コマンドやってくれること
/commit変更内容を見て、適切なメッセージでGitコミット
/daily-scheduleその日の工程表を自動作成
/daily-report日報を自動作成
/sales-dm営業DM文面を作成
/meeting-notes議事録をHTML形式で作成

💡 自分でスキルを作ることもできる。「毎回やってること」があればスキル化すると楽になる。

3サブエージェント — 分身して同時作業

大きなタスクを「子分」に分割して、複数の作業を同時に進められる。

普通の作業
タスクA → タスクB → タスクC
順番に1つずつ → 時間かかる
サブエージェント使用
タスクA ─→ 完了
タスクB ─→ 完了 （同時進行）
タスクC ─→ 完了
→ 結果を統合して納品
こんなときに勝手に使ってくれる
・「10ページのサイト作って」
・「このコードのバグ全部直して」
・「3つのファイルを同時にリサーチ」
・「テストを走らせながら別の修正」

指示する必要なし。AIが自動で判断して分身する。

4MCP — 外部サービスと直接連携

Claude Code から他のサービスを直接操作できる仕組み。

📅

セットアップ & HTML活用

社内資料をHTMLにする理由

この研修資料自体が、その実例です。

比較ポイントWord / PDFHTML（GitHub Pages）
共有方法ファイルを送付・DLURLを送るだけ
バージョン管理「最新版_v3_final.docx」問題常に最新が1つ。更新即反映
マルチデバイスPC以外で崩れがちスマホ・タブレットOK
AI作成・修正Wordの操作はAIが苦手エージェントが直接書ける
デザイン自由度テンプレート依存制限なし。ブランドに合わせ放題
コストOffice ライセンス必要完全無料（GitHub Pages）

つまり: 「AIに資料を作ってもらう → URLで全員に共有 → 修正もAIに頼む」
このワークフローが回るのはHTMLだけ。

ドキュメント形式の HTML

スライドだけじゃない。Wordのような縦スクロール文書も作れる。

Wordと同じ読み心地
縦スクロールで上から読む。見出し・表・箇条書き、全部使える。
印刷・PDF化もOK
ブラウザの印刷機能でそのままキレイに出力できる。
AIが直接作成
